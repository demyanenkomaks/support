# Тесты валидаторов

## Описание

Этот набор тестов проверяет корректность работы всех валидаторов пакета `maksde/support`.

## Установка зависимостей

Перед запуском тестов необходимо установить зависимости:

```bash
cd vendor/maksde/support
composer install
```

## Структура тестов

```
tests/
└── Unit/
    ├── EmailValidateTest.php            # Тесты валидации почты
    ├── PhoneValidateTest.php            # Тесты валидации российского телефона
    ├── PhoneInternationalValidateTest.php # Тесты валидации международного телефона
    ├── NameValidateTest.php             # Тесты валидации имени/фамилии/отчества
    ├── QuantityValidateTest.php         # Тесты валидации количества
    ├── DateValidateTest.php             # Тесты валидации даты
    ├── TimeValidateTest.php             # Тесты валидации времени
    ├── DateTimeValidateTest.php         # Тесты валидации даты и времени
    ├── ImageValidateTest.php            # Тесты валидации изображений
    ├── VideoValidateTest.php            # Тесты валидации видео
    └── DocumentValidateTest.php         # Тесты валидации документов
```

## Запуск тестов

### Все тесты

```bash
composer test
```

или

```bash
./vendor/bin/phpunit
```

### Конкретный тест

```bash
./vendor/bin/phpunit tests/Unit/EmailValidateTest.php
```

### С отчетом о покрытии кода

```bash
composer test:coverage
```

Отчет будет сгенерирован в папке `coverage/`.

### Запуск одного метода теста

```bash
./vendor/bin/phpunit --filter test_valid_emails
```

## Покрытие тестами

Каждый валидатор имеет тесты для:
- ✅ Валидных значений (должны проходить валидацию)
- ❌ Невалидных значений (должны проваливать валидацию)

### EmailValidateTest

**Валидные значения:**
- Простой email: `test@example.com`
- С поддоменом: `user@mail.example.com`
- С цифрами: `user123@example.com`
- С точками: `first.last@example.com`
- С кириллическим доменом: `test@домен.рф`

**Невалидные значения:**
- Без символа @: `testexample.com`
- Несколько @: `test@@example.com`
- Без домена: `test@`
- Слишком длинный email

### PhoneValidateTest

**Валидные значения:**
- Стандартный формат: `+71234567890`
- С пробелами по краям: ` +71234567890 `

**Невалидные значения:**
- Без плюса: `71234567890`
- Неверный код страны: `+81234567890`
- С пробелами внутри: `+7 123 456 78 90`
- С дефисами: `+7-123-456-78-90`

### PhoneInternationalValidateTest

**Валидные значения:**
- Минимальная длина (7 цифр): `+1234567`
- Максимальная длина (15 цифр): `+123456789012345`
- Форматы разных стран: `+12025551234`, `+442071234567`

**Невалидные значения:**
- Без плюса: `1234567890`
- Слишком короткий: `+123456`
- Слишком длинный: `+1234567890123456`
- С форматированием: `+1-202-555-1234`

### NameValidateTest

**Валидные значения:**
- Кириллица: `Алексей`
- Латиница: `John`
- С дефисом: `Жан-Клод`
- С пробелом: `Анна Мария`
- С апострофом: `O'Connor`

**Невалидные значения:**
- С цифрами: `Иван123`
- Со спецсимволами: `Иван@`
- Только пробелы: `   `
- Слишком длинное имя (>50 символов)

### QuantityValidateTest

**Валидные значения:**
- Ноль: `0`
- Положительное целое: `10`, `100`
- Большое число: `999999`

**Невалидные значения:**
- Отрицательное: `-1` (с min по умолчанию)
- Дробное: `10.5`
- С символом e: `1e5`
- С плюсом/минусом: `+10`, `-10`

**Проверка пользовательских ограничений:**
- `min: 10` - проверка, что значения >= 10 проходят, а < 10 - нет
- `max: 100` - проверка, что значения <= 100 проходят, а > 100 - нет
- `min: 10, max: 100` - проверка диапазона [10, 100]
- `min: null, max: 100` - проверка, что отрицательные значения допустимы
- Проверка исключения при min > max

### DateValidateTest

**Валидные значения:**
- Стандартная дата: `2025-04-16`
- Високосный год: `2024-02-29`
- С пробелами по краям: ` 2025-04-16 `

**Невалидные значения:**
- Неверный формат: `16.04.2025`, `04/16/2025`
- Несуществующая дата: `2025-13-01`, `2025-02-29`
- Неполная дата: `2025-04`

**Проверка ограничений по времени:**
- `'future'` - проверка, что будущие даты проходят, а прошлые и текущая - нет
- `'past'` - проверка, что прошлые даты проходят, а будущие и текущая - нет
- `null` - проверка, что любые даты (прошлые, текущая, будущие) проходят
- Проверка исключения при недопустимом значении параметра

### TimeValidateTest

**Валидные значения:**
- Полночь: `00:00:00`
- Полдень: `12:00:00`
- Конец дня: `23:59:59`

**Невалидные значения:**
- Без секунд: `14:30`
- Невалидные значения: `24:00:00`, `14:60:00`, `14:30:60`
- Буквы: `ab:cd:ef`

**Проверка ограничений по времени:**
- `'future'` - проверка, что будущее время проходит, а прошлое - нет
- `'past'` - проверка, что прошлое время проходит, а будущее - нет
- `null` - проверка, что любое время (прошлое, текущее, будущее) проходит
- Проверка исключения при недопустимом значении параметра

### DateTimeValidateTest

**Валидные значения:**
- Стандартный формат: `2025-04-16 14:30:45`
- Полночь: `2025-04-16 00:00:00`
- Високосный год: `2024-02-29 12:00:00`

**Невалидные значения:**
- С символом T: `2025-04-16T14:30:45`
- Только дата: `2025-04-16`
- Только время: `14:30:45`
- Невалидные компоненты: `2025-13-01 14:30:45`, `2025-04-16 25:30:45`

**Проверка ограничений по времени:**
- `'future'` - проверка, что будущие дата и время проходят, а прошлые - нет
- `'past'` - проверка, что прошлые дата и время проходят, а будущие - нет
- `null` - проверка, что любые дата и время (прошлые, текущие, будущие) проходят
- Проверка исключения при недопустимом значении параметра

### ImageValidateTest

**Валидные значения:**
- JPG/JPEG файлы с корректным MIME-типом
- PNG файлы с корректным MIME-типом
- HEIC и WebP файлы
- Файлы размером до 10 МБ

**Невалидные значения:**
- Файлы с неразрешенным расширением (GIF, BMP, SVG)
- Файлы с неверным MIME-типом
- Файлы размером больше 10 МБ
- Не файловые значения (пропускаются)

**Проверка переопределения параметров:**
- Пользовательские расширения
- Пользовательские MIME-типы
- Пользовательский максимальный размер

### VideoValidateTest

**Валидные значения:**
- MP4 файлы с корректным MIME-типом
- WebM файлы с корректным MIME-типом
- HEVC файлы
- Файлы размером до 20 МБ

**Невалидные значения:**
- Файлы с неразрешенным расширением (AVI, MOV, FLV)
- Файлы с неверным MIME-типом
- Файлы размером больше 20 МБ
- Не файловые значения (пропускаются)

**Проверка переопределения параметров:**
- Пользовательские расширения
- Пользовательские MIME-типы
- Пользовательский максимальный размер

### DocumentValidateTest

**Валидные значения:**
- DOCX файлы с корректным MIME-типом
- XLSX файлы с корректным MIME-типом
- PDF файлы
- Файлы размером до 10 МБ

**Невалидные значения:**
- Файлы с неразрешенным расширением (DOC, XLS, TXT)
- Файлы с неверным MIME-типом
- Файлы размером больше 10 МБ
- Не файловые значения (пропускаются)

**Проверка переопределения параметров:**
- Пользовательские расширения
- Пользовательские MIME-типы
- Пользовательский максимальный размер

## Добавление новых тестов

Чтобы добавить новый тест:

1. Создайте новый файл в `tests/Unit/`
2. Наследуйте класс от `PHPUnit\Framework\TestCase`
3. Используйте data providers для предоставления тестовых данных
4. Запустите тесты для проверки

Пример:

```php
<?php

namespace Maksde\Support\Tests\Unit;

use PHPUnit\Framework\TestCase;

class MyValidateTest extends TestCase
{
    /**
     * @dataProvider validDataProvider
     */
    public function test_valid_data($data): void
    {
        $fails = false;
        $validator = new MyValidate;
        
        $validator->validate('field', $data, function () use (&$fails) {
            $fails = true;
        });

        $this->assertFalse($fails);
    }

    public static function validDataProvider(): array
    {
        return [
            'case 1' => ['value1'],
            'case 2' => ['value2'],
        ];
    }
}
```

## Требования

- PHP >= 8.2
- PHPUnit ^11.0
- Composer

## Примечания

- Все тесты используют data providers для организации тестовых данных
- Моки функции `config()` используются там, где валидаторы зависят от конфигурации
- Тесты изолированы и не зависят от внешних сервисов
