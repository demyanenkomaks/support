# Анализ валидации Email по PATTERN

## 📋 Текущее регулярное выражение

```regex
/^(?!\.)(?!.*\.\.)([A-Za-zА-Яа-яЁё0-9_'+\-\.]*)([A-Za-zА-Яа-яЁё0-9_+-])@([A-Za-zА-Яа-яЁё0-9]([A-Za-zА-Яа-яЁё0-9-]*[A-Za-zА-Яа-яЁё0-9])?\.)+[A-Za-zА-Яа-яЁё]{2,}$/u
```

## 🔍 Правила валидации

### 1. Локальная часть (до @)

**Разрешенные символы:**
- Латинские буквы: `A-Z`, `a-z`
- Кириллические буквы: `А-Я`, `а-я`, `Ё`, `ё`
- Цифры: `0-9`
- Специальные символы: `_` (подчеркивание), `'` (апостроф), `+` (плюс), `-` (дефис), `.` (точка)

**Ограничения:**
- НЕ должна начинаться с точки: `(?!\.)`
- НЕ должна содержать две точки подряд: `(?!.*\.\.)`
- НЕ должна заканчиваться точкой: последний символ `([A-Za-zА-Яа-яЁё0-9_+-])` исключает точку
- Максимальная длина: **64 символа**

**Важные особенности:**
- ✅ РАЗРЕШЕНО начинаться с дефиса: `-user@domain.com`
- ✅ РАЗРЕШЕНО начинаться с подчеркивания: `_user@domain.com`
- ✅ РАЗРЕШЕНО заканчиваться дефисом: `user-@domain.com`
- ✅ РАЗРЕШЕНО заканчиваться подчеркиванием: `user_@domain.com`
- ✅ РАЗРЕШЕНО заканчиваться плюсом: `user+@domain.com`
- ✅ РАЗРЕШЕНЫ двойные дефисы: `user--name@domain.com`
- ✅ РАЗРЕШЕНЫ двойные подчеркивания: `user__name@domain.com`

### 2. Доменная часть (после @)

**Разрешенные символы:**
- Латинские буквы: `A-Z`, `a-z`
- Кириллические буквы: `А-Я`, `а-я`, `Ё`, `ё`
- Цифры: `0-9`
- Дефис: `-` (только в середине)

**Ограничения:**
- Каждая часть домена (между точками) ДОЛЖНА:
  - Начинаться с буквы или цифры: `[A-Za-zА-Яа-яЁё0-9]`
  - Заканчиваться буквой или цифрой: `[A-Za-zА-Яа-яЁё0-9]`
  - Может содержать дефисы только в середине: `[A-Za-zА-Яа-яЁё0-9-]*`
- ❌ ЗАПРЕЩЕНО: подчеркивание (`_`), плюс (`+`), любые другие символы
- ✅ РАЗРЕШЕНЫ двойные дефисы в середине: `user@my--domain.com`
- Максимальная длина: **253 символа**

### 3. TLD (доменная зона)

**Разрешенные символы:**
- ТОЛЬКО буквы (латиница или кириллица): `[A-Za-zА-Яа-яЁё]{2,}`
- Минимум 2 символа

**Дополнительная проверка:**
- Валидация через `DomainZone::isValidTLD()` (1317 существующих TLD)

### 4. Общие ограничения

- Максимальная общая длина: **255 символов**
- НЕ должно быть двух точек подряд в любом месте: `(?!.*\.\.)`

### 2. Проблемы с доменом (9)

```
user@do+main.com                    # Плюс в домене ЗАПРЕЩЕН

userdomain.com                      # Нет @
@domain.com                         # Нет локальной части
user@                               # Нет домена
user@@domain.com                    # Двойной @@
user@my@domain.com                  # @ в домене
user@-domain.com                    # Домен начинается с дефиса
user@.domain.com                    # Домен начинается с точки
```

### 3. Подчеркивание в домене (6)

```
user@my__domain.com                 # Подчеркивание в домене ЗАПРЕЩЕНО
user@my._domain.com                 # Подчеркивание в домене ЗАПРЕЩЕНО
user@my_-domain.com                 # Подчеркивание в домене ЗАПРЕЩЕНО
user@my...domain.com                # Три точки подряд
user@my._-domain.com                # Подчеркивание в домене ЗАПРЕЩЕНО
user@_domain.com                    # Домен начинается с подчеркивания
```

### 4. Проблемы с точками (9)

```
.user@domain.com                    # Начинается с точки
user.@domain.com                    # Локальная часть заканчивается точкой
user@domain.com.                    # Заканчивается точкой
user..name@domain.com               # Две точки подряд в локальной части
user...name@domain.com              # Три точки подряд в локальной части
user@my..domain.com                 # Две точки подряд в домене
user@my...domain.com                # Три точки подряд в домене
user@my.-domain.com                 # Домен начинается с дефиса после точки
.user.@domain.com                   # Начинается и заканчивается точкой
```

### 5. Проблемы с дефисами и подчеркиваниями (7)

```
user@domain-.com                    # Часть домена заканчивается дефисом
user@domain.com-                    # TLD заканчивается дефисом
user@domain_.com                    # Подчеркивание в домене ЗАПРЕЩЕНО
user@domain._com                    # Подчеркивание перед TLD
user@domain.com_                    # TLD заканчивается подчеркиванием
user@domain.co-m                    # Дефис в TLD ЗАПРЕЩЕН
user@domain.co_m                    # Подчеркивание в TLD ЗАПРЕЩЕНО
```
